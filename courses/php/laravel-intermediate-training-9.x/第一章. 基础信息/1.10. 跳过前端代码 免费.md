# 1.10. 跳过前端代码 免费

原文链接：https://learnku.com/courses/laravel-intermediate-training/9.x/skip-front-end-code/12545

## 说明

Laravel Mix 为我们提供了功能齐全的前端工作流。

然而，我收到许多新人反馈，卡在『前端编译』上，花费了很长时间，都无法通过。最后不得已放弃学习。

从反馈中发现，『卡住』的原因多种多样。有的因为电脑硬件、有的是系统的不同、有的甚至是软件的设置不一致，再加上有些同学有可能接触编程不久，不具备调试能力。对于作者我来说，很难提供一套『完整的解决方案』。

因前端工作流而放弃学习，让我感到心疼。所以这里提供一个方法，来暂时跳过『前端工作流』，让你的学习能继续下去。

如果你在这个问题上已经花费了几个小时，始终无法解决，不要犹豫，直接使用此方法。

科技行业有句话：

>

Done is better than perfect.

先完成课程，多做几遍，届时有足够的自信了，再来解决这个问题。

下面是具体的操作。

## 1. 下载文件并替换

前往 [代码仓库](https://learnku.com/courses/laravel-essential-training/9.x/the-book-source-code) 下载源码。

将下载的以下源码文件放入（或替换）对应的文件夹内：

| 文件 |
| --- |
| 说明 |

| public/css/app.css |
| --- |
| 样式文件 |

| public/js/app.js |
| --- |
| 前端脚本文件 |

| public/mix-manifest.json |
| --- |
| 模板中使用 `mix()` 时，函数内部会用到这个文件 |

## 2. 后续注意

咱们上面替换的文件，是项目最终的 JS 和 CSS 文件，自此以后，无需运行:

```
$ npm run watch-poll
```

或：

```
$ npm run dev
```

这些命令会重新生成上面我们替换的文件。

`resources` 下的 .scss 或者 .js 文件还是可以修改的，跳过上面提到的命令即可。
