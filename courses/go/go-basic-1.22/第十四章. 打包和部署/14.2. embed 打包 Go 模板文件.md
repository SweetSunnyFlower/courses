# 14.2. embed 打包 Go 模板文件

原文链接：https://learnku.com/courses/go-basic/1.22/packaging-go-programs-with-embedded/16559

## 说明

本节我们来讲解如何将 Goblog 应用打包为一个单独的可执行文件（包括 CSS/JS 和 模板文件）。

>

这一节有视频版本的，请见 [007. 标准库 embed 包：使用 embed 打包模板和静态文件](https://learnku.com/courses/go-video/2022/007-use-embed-to-package-templates-and-static-files/11574)

## 计划

我们会先打包模板，然后再打包 CSS/JS 文件。最终结果将会是一个完整的可执行文件。

需要说明的是，`.env` 文件并没有打包。这是因为环境变量配置信息（数据库连接、端口、网站 URL 等）会随着服务器而变动，没有打包的必要。

## embed 类型

embed 支持三种嵌入类型：

- string —— 加载单个文件，并将内容作为字符串使用（utf8 编码）

- byte slice ( `[]byte` 字节数组 )—— 将单个文件以二进制格式加载

- embed.FS —— 以文件系统方式加载目录和子目录文件树

嵌入的内容都是只读的。也就是在编译期嵌入文件的内容是什么，那么在运行时的内容也就是什么。

接下来的文章里，我们将使用文件系统的嵌入类型 ——  `embed.FS`。

`embed.FS` 文件系统提供了打开和读取的方法：

```
type FS
func (f FS) Open(name string) (fs.File, error)
func (f FS) ReadDir(name string) ([]fs.DirEntry, error)
func (f FS) ReadFile(name string) ([]byte, error)
```

其本质是 `embed.FS` 实现了 `io/fs` 接口。意味着我们可以把他当正常的文件系统使用，例如说使用 `fs.Sub()` 来读取子目录。当然所有的操作都是只读。

## embed 指令简介

`embed.FS` 的加载语法：

```
package main

import (
"embed"
"fmt"
)

//go:embed public/*
var staticFS embed.FS

func main() {
data, _ := staticFS.ReadFile("public/css/app.css")
fmt.Println(string(data))
}
```

以上会打印出来 `public/css/app.css` 里的内容。

```
//go:embed public/*
var staticFS embed.FS
```

`//go:embed` 我们称之为指令，后面跟着文件目录，或者是单个文件，支持通配符。`public/*` 意味着递归加载 public 下所有子目录以及文件。

知道下面这几个点可避免使用 embed.FS 时产生困惑：

- embed.FS 就是一个打包在二进制包里的文件系统；

- `//go:embed` 指令是告知要加载哪些文件目录；

- 加载和读取目录时，会 保持原有的目录结构；

- 除了写功能，你可以将其当做一般文件系统来读取里面的文件和目录。

## 打包模板

embed 读取相对路径是相对于书写 `//go:embed` 指令的 .go 文件的。

要加载根目录下的 `resources` 目录，在 `pkg/view/view.go` 文件里书写  `//go:embed` 指令将 报错无法找到 `resources` 文件。

我们只能在 `resources` 同级目录中，也就是根目录下的 main.go 中设置 embed 指令变量，并将变量传递到 `pkg/view/view.go` 里。

main.go

```
package main

import (
"embed"
"goblog/app/http/middlewares"
"goblog/bootstrap"
"goblog/config"
c "goblog/pkg/config"
"goblog/pkg/logger"
"net/http"
)

//go:embed resources/views/articles/*
//go:embed resources/views/auth/*
//go:embed resources/views/categories/*
//go:embed resources/views/layouts/*
var tplFS embed.FS

func init() {
// 初始化配置信息
config.Initialize()
}

func main() {
// 初始化 SQL
bootstrap.SetupDB()

// 初始化模板
bootstrap.SetupTemplate(tplFS)

// 初始化路由绑定
router := bootstrap.SetupRoute()

err := http.ListenAndServe(":"+c.GetString("app.port"), middlewares.RemoveTrailingSlash(router))
logger.LogError(err)
}
```

我们先来看这一段：

```
//go:embed resources/views/articles/*
//go:embed resources/views/auth/*
//go:embed resources/views/categories/*
//go:embed resources/views/layouts/*
var tplFS embed.FS
```

tplFS 变量加载了我们的模板目录，可以看到 `//go:embed` 指令支持设置加载多个目录，依次叠加即可，注意中间不能有空行。

为什么不直接使用：

```
//go:embed resources/*
var tplFS embed.FS
```

在使用中发现这种加载方式无法加载以下划线开头的文件，如 `_article_meta.gohtml`。

>

感兴趣的同学可以前往查看这个调试视频 —— [008. 调试 Go 代码：使用 VSCode 断点调试 Go 程序](https://learnku.com/courses/go-video/2022/debugging-go-programs-using-vscode-breakpoints/11575) 。

在 main 函数里，我们新增了初始化模板的调用，将我们的 `tplFS` 传参：

```
// 初始化模板

```

`bootstrap.SetupTemplate` 还不存在，我们来创建：

### 测试一下

运行：

```
$ go build -o goblog
```

将模板目录删除（不怕，我们有代码版本控制护航），手动删除或者使用以下命令：

```
$ rm -rf resources
```

运行我们的 Goblog 程序：

```
$ ./goblog
```

打开浏览器访问我们的应用 [localhost:3000](http://localhost:3000) 可以看到模板能正常渲染，这意味着我们的 goblog 是从二进制的 embed 文件系统中加载模板。

最后将我们上面删除的目录还原回来：

```
$ git checkout resources
```
