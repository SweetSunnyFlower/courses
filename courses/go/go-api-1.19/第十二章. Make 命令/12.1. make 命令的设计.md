# 12.1. make 命令的设计

原文链接：https://learnku.com/courses/go-api/1.19/design-of-make-command/13540

## 说明

本节讲解下 make 命令需求。

## Make 命令

下面这个 Gohub 项目的所有 make 子命令：

1. make model —— 生成模型文件

2. make apicontroller —— 生成 Restful API 控制器

3. make request —— 生成请求验证文件

4. make policy —— 生成授权文件

5. Make cmd —— 生成自定义命令文件

6. make migration —— 生成数据迁移文件

7. make seeder —— 生成数据填充文件

8. make factory —— 生成工厂模式

1-5 是我们这一章要开发的，6、7、8 我们等到用到时再来开发。

## 生成代码

因为我们现在的开发，都遵循着『约定俗成』的开发惯例，这跟 Laravel 或 Ruby on Rails 框架很类似。

模型、控制器、验证请求、请求授权等这些机制，不同模型间的代码都基本类似。这也为我们自动生成代码提供了可能。

看下 User 模型：

```go
// User 用户模型
type User struct {
    models.BaseModel
    ...
    models.CommonTimestampsField
}

// Create 创建用户，通过 User.ID 来判断是否创建成功
func (userModel *User) Create() {
    database.DB.Create(&userModel)
}

func (userModel *User) Save() (rowsAffected int64) {
    result := database.DB.Save(&userModel)
    return result.RowsAffected
}
```

如果我们有一个 Category 模型，将以上代码中的 User 替换为 Category，userModel 替换为 categoryModel，再使用小写单数的 category_model.go 命名，就生成了一个新的  Category 模型文件了。

## 流程

还是以 make model 为例：

1. 创建一个 model.stub 的模板文件，里面写好变量（如 PackageName，StructName 等）；

2. 读取用户传参的模型名称参数，如 `make model user` ,`user` 就是我们需要的模型名称参数；

3. 获取到 `user` 的值后，对其做格式化，例如说 VariableName（单数变量 user）、VariableNamePlural（变量复数 users）;

4. 调用 Go 的 `strings.ReplaceAll` 方法，将模板中的变量进行替换；

5. 存入文件，按照一定的命名规范；

6. make model 命令执行完成。

不止是模型文件，大部分类型的文件，都可使用上面的流程来生成。
