# 20.2. 后面的学习建议

原文链接：https://learnku.com/courses/go-api/1.19/learning-suggestions/13602

## 学习建议

有几点学习的建议给大家：

1. 按照本书，至少把项目 做 5 遍，在实战中记忆框架的功能，扎实的基本功对于以后的技能成长至关重要；

2. 挑战你自己，尝试使用文字将项目的功能写个列表，然后在不看教程的情况下实现整个项目的功能；

## 扩展本项目

本项目源码采用 MIT 开源，你可以在此基础上扩展功能。

或者找一些优秀的开源包，将其集成到项目中。

目前我们的系统中，cache 包仅支持 redis，可尝试使用其他缓存方案实现，只需要实现 Store interface 即可。

另外 verifycode 、mail、sms 都可以继续扩展，可以尝试扩展他们。

目的是将自己埋到代码里面，只有不断编码，且追求优质的代码，才能精进。

## 开源

课程中的 pkg 包，或者以后项目开发中实现的的技术方案，都可考虑将其整理起来，开源到 Github 等开源平台上。

开源可以提高对代码品质的要求，让我们写更好的代码。

简历上有开源项目，求职时也会更轻松点，大部分公司，尤其是大公司，都希望招聘有优秀开源项目的员工。

## 阅读源码

Gohub 使用到很多开源项目，这些项目都推荐去看下他们的源码，尝试从源码来推断出他们的『使用文档』，也可以适当的做中文注释，以加深印象。

阅读源码与写代码一样重要，尤其是阅读优秀的开源项目源码。要学会站在巨人的肩膀上，优秀的开源项目，肯定会有很多可以借鉴的地方。

优秀的代码阅读多了，再加上不断实践，与大师同行，注定优秀。

## 写作

作为一名工程师，写作能力是一项必要技能。在工作中，开发文档的撰写、技术方案的出具，都需要用到此技能。

写作能力出众的工程师通常具有逻辑清晰，沟通能力强等特质。

写作是一门技能，同编程一样，需要大量练习才能掌握。建议给自己设定要求，每周在 [Learnku Go 社区上](https://learnku.com/go) 发布两篇以上文章，内容可以是：

- 学习此课程的笔记；

- 其他技术话题学习后的总结；

- 编程心得和解决方案记录等。

在社区书写文章能收到 Go 从业者的点赞和评论，收到赞誉对写作积极性会有所帮助，也请你不吝为他人点赞。评论能让大家交换知识，有时候你会收到修改建议，有时候你会收获到针对某个问题的更多解决方案。

请明白，写作的积累，对以后的 [求职](https://learnku.com/go/c/jobs) 会有非常大的帮助，因为大部分用人单位会优先录用写作能力出众的工程师。

把写作当成简历的一部分来写。
