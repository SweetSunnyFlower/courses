# 4.3. 用户模型

原文链接：https://learnku.com/courses/go-api/1.19/user-model/13490

## 说明

这节课我们先来创建用户模型。

## 1. 创建模型基类

基类文件用以存放模型通用属性和方法。

创建文件：

app/models/model.go

```
// Package models 模型通用属性和方法
package models

import (
"time"
)

// BaseModel 模型基类
type BaseModel struct {
ID uint64 `gorm:"column:id;primaryKey;autoIncrement;" json:"id,omitempty"`
}

// CommonTimestampsField 时间戳
type CommonTimestampsField struct {
CreatedAt time.Time `gorm:"column:created_at;index;" json:"created_at,omitempty"`
UpdatedAt time.Time `gorm:"column:updated_at;index;" json:"updated_at,omitempty"`
}
```

## 2. 创建 User 模型

模型文件里存放的是模型定义，以及对象操作的逻辑代码。

app/models/user/user_model.go

```
// Package user 存放用户 Model 相关逻辑
package user

import (
"gohub/app/models"
)

// User 用户模型
type User struct {
models.BaseModel

Name     string `json:"name,omitempty"`
Email    string `json:"-"`
Phone    string `json:"-"`
Password string `json:"-"`

models.CommonTimestampsField
}
```

注意，因为我们不希望将敏感信息输出给用户，所以这里 Email 、Phone 、Password 后面设置了 `json:"-"` ，这是在指示 JSON 解析器忽略字段 。后面接口返回用户数据时候，这三个字段都会被隐藏。

## 3. 创建 user_util.go 文件

user_util.go 文件里存放的是模型相关的数据库操作。

可以直接使用 `user.` 调用的，都存放于此文件中：

```
user.IsEmailExist(email)
```

在大项目中，随着项目的开发，业务逻辑的增多，后面模型文件会变得很臃肿，难以维护。提早划分文件，可以有效避免此问题。

创建文件：

app/models/user/user_util.go

```
package user

import (
"gohub/pkg/database"
)

// IsEmailExist 判断 Email 已被注册
func IsEmailExist(email string) bool {
var count int64
database.DB.Model(User{}).Where("email = ?", email).Count(&count)
return count > 0
}

// IsPhoneExist 判断手机号已被注册
func IsPhoneExist(phone string) bool {
var count int64
database.DB.Model(User{}).Where("phone = ?", phone).Count(&count)
return count > 0
}
```
